release notes

taskly 0.5.2 (current release)

  • the lato font has been fixed on webkit browsers.

taskly 0.5.1

  • music and quick links widgets added.
  • load button added to quick notes.

taskly 0.5

  • taskly features are now fully functional on webkit browsers! we are working on getting the lato font working on webkit but helvetica is being used as a placeholder for now.
  • quick notes and a grade calculator have been added. please report any issues with either one through the feedback form.
  • we are still working on the taskly mobile website. taskly is only partially functional on mobile. timers and alarms do not work. additionally, it does not render properly on smaller devices.
  • because of browser features that force some websites to go idle when the tab is not selected, some timers and alarms may be inaccurate. we are investigating this issue and will have a fix soon.
  • we fixed issues that allowed null alarms and timers to be created.
  • please use the feedback form to report any other issues!

taskly 0.4

  • many taskly features are still unavailable or glitchy on webkit-based browsers. however, many improvements have been made and will keep coming in future updates. we will have taskly running on webkit-based browsers smoothly by the first stable update.
  • task lists are now available on taskly. the client that manages task lists is generally stable and list items are saved to the browser's local storage.
  • because of browser features that force some websites to go idle when the tab is not selected, some timers and alarms may be inaccurate. we are investigating this issue and will have a fix soon.
  • many javascript features of taskly simply do not work on mobile devices. we are working to correct this but please note that taskly is still not reliable and does not render properly on mobile devices. we are working on a mobile version that the taskly website would automatically redirect to on mobile devices.
    • note: we have identified the issue as an incompatibility with the notifications api on mobile devices. the problem will be resolved when taskly mobile is released.
  • please use the feedback form to report any other issues!

taskly 0.3

  • many taskly features are still unavailable or glitchy on webkit-based browsers. however, many improvements have been made and will keep coming in future updates. we will have taskly running on webkit-based browsers smoothly by the first stable update.
  • alarms are now available on taskly. there are currently no known issues but if one is found please report it using the feedback form.
  • many javascript features of taskly simply do not work on mobile devices. we are working to correct this but please note that taskly is not reliable and does not render properly on mobile devices.

taskly 0.2

  • the preferred font for taskly's user interface is still not working on webkit-based browsers. we are working to fix this issue. however, we have made sure that fonts are consistent even when the #1 font is not available.
  • timers are now fully functional on taskly. timer names should be working but may show up as undefined; if they do, please contact us using the feedback form so that we can identify the issue.
  • taskly is a beta product and has not been optimized for mobile devices yet. therefore, there will be website rendering issues on some smaller mobile devices.
  • the taskly notification manager is being used to announce when timers have elapsed. we are working to make a popup menu on the website as well; this will be available in the very next release.
